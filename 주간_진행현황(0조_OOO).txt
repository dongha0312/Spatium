 * 금주 진행 사항(상세히 작성)
  > 로그인/로그아웃 API 구현 (백엔드)
    - JWT 기반 인증 도입 (jjwt 라이브러리 추가, JwtUtil 클래스 구현 : 토큰 생성/검증)
    - 일반 로그인 API 구현 (POST /api/auth/sessions)
      · 이메일/비밀번호 검증 후 accessToken(1시간)/refreshToken(14일) 발급
      · 실패 시 401 INVALID_CREDENTIALS 응답 (명세서 에러 형식 준수)
    - 로그아웃 API 구현 (DELETE /api/auth/sessions/current)
      · Authorization 헤더의 토큰 검증 후 204 No Content 반환
      · 유효하지 않은 토큰 시 401 UNAUTHORIZED 응답
    - 소셜 로그인에도 JWT 토큰 발급 적용 (일반 로그인과 동일한 응답 구조로 통일)
    - 로그인/로그아웃 관련 DTO 추가 (LoginRequest, LoginResponse, UserSummaryResponse)

  > 프론트엔드 연동 (React)
    - 로그인 페이지를 백엔드 API와 연동 (기존 데모 처리 → 실제 인증으로 교체)
    - 로그인 성공 시 JWT 토큰을 localStorage에 저장하도록 세션 저장소 확장
    - 마이페이지 로그아웃 버튼을 로그아웃 API와 연동 (호출 후 세션 삭제)
    - 구글 소셜 로그인 성공 시에도 토큰 저장 처리

  > 버그 수정
    - 회원가입 API 404 오류 수정 (트레일링 슬래시 경로 문제)
    - 소셜 로그인 시 가입된 회원인데 회원가입 페이지로 이동하는 버그 수정
      (컨트롤러 경로 불일치 /api → /api/auth 정리)

  > 테스트
    - 로그인 : 정상 200 + 토큰 발급 / 틀린 비밀번호·미가입 이메일 401 확인
    - 로그아웃 : 정상 토큰 204 / 위조 토큰·헤더 누락 401 확인
    - 브라우저에서 로그인 → 토큰 저장 → 로그아웃 전체 흐름 검증 완료
